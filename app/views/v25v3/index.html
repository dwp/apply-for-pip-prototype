Iteration 2

Tested - Oct 2025

Changes based on UR findings:

  • Making the content clearer when telling a user to enter one condition at a time
  • Content improvements to tell the user what to do if their condition is undiagnosed
  • Most users misunderstood they could enter their condition manually into the same field in iteration 1. This iteration includes additional hint text and change to the drop down design, leaving a gap between the input field and adding the title of 'Suggestions', a pattern used by DofE.

In this UR round we tested 2 designs:

  • Option A - testing the raw RDS list of health conditions with no changes
  • Option B - testing the RDS list of health conditions with content improvements


Iteration 1

Tested - Sep 2025

All designs in this iteration include replacing the health conditon free text field with a typeahead component and using the RDS list. If the user cannot find their condition in the standardised list, they can enter their condition manually in the same field.

In this UR round we tested 3 designs:

  • Option A - testing the raw RDS list of health conditions with no changes
  • Option B - testing the RDS list of health conditions with content improvements
  • Option C - testing 'pop-up' content to tell a user they can enter their condition manually
